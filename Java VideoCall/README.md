# Java Видеозвонки (UI Angular + Jitsi Meet)

Реализация видеоконференций для занятий с использованием **Jitsi Meet** (self-hosted) и **JWT авторизации**

**Стек:**
- NodeJs `v24.13.1`
- Angular `v21.1.4`
- Java `v21`
- Gradle `v8.12`
- PostgreSQL `v15`
- Jitsi Meet `stable-10741`

---

## Настройка проекта

### Сеть

**Backend:**
- Локальный URL: `http://localhost:8081`
- Публичный URL: `https://angular.cloudpub.ru/api` (через reverse proxy)
- Публикация по HTTP, порт `8081`

**Frontend:**
- Локальный URL: `http://localhost:4200`
- Публичный URL: `https://angular.cloudpub.ru`
- Публикация по HTTPS (порт 443, reverse proxy)

**Jitsi Meet (Docker):**
- Локальный URL: `http://localhost:8000`
- Публичный URL: `https://200.87.154.47:8443` (прямой доступ через Tailscale)
- Порты: `8000` (HTTP), `8443` (HTTPS), `10000/udp` (видео)

**Важно:** 
- Backend и Frontend должны быть в одном домене (`.cloudpub.ru`) для работы cookie
- Jitsi может быть на отдельном домене/IP (используется JWT для авторизации)

**БД:** PostgreSQL, локальный URL: `http://localhost:5433`

## БД

**Название:** `logoped_db`

### Таблицы

```sql
-- Пользователи (логопеды и пациенты)
CREATE TABLE public.users (
    id int8 GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username varchar(50) UNIQUE NOT NULL,
    password varchar(255) NOT NULL,
    role varchar(20) NOT NULL, -- 'LOGOPED' или 'PATIENT'
    full_name varchar(100),
    logoped_id int8, -- ссылка на логопеда (для пациентов)
    CONSTRAINT fk_patient_logoped FOREIGN KEY (logoped_id) REFERENCES users(id)
);

-- Занятия
CREATE TABLE public.session (
    id int8 GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title varchar(255) NOT NULL,
    start_time timestamptz NOT NULL,
    end_time timestamptz NOT NULL,
    status varchar(20) NOT NULL, -- 'SCHEDULED', 'ACTIVE', 'COMPLETED'
    logoped_id int8 NOT NULL,
    room_name varchar(100) UNIQUE NOT NULL, -- имя комнаты Jitsi
    CONSTRAINT fk_session_logoped FOREIGN KEY (logoped_id) REFERENCES users(id)
);

-- Участники занятия (пациенты)
CREATE TABLE public.session_patients (
    session_id int8 NOT NULL,
    patient_id int8 NOT NULL,
    PRIMARY KEY (session_id, patient_id),
    CONSTRAINT fk_sp_session FOREIGN KEY (session_id) REFERENCES session(id),
    CONSTRAINT fk_sp_patient FOREIGN KEY (patient_id) REFERENCES users(id)
);

-- Материалы занятия (презентации, документы)
CREATE TABLE public.session_material (
    id int8 GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    session_id int8 NOT NULL,
    file_name varchar(255) NOT NULL,
    file_path varchar(500) NOT NULL,
    file_type varchar(20), -- 'PPTX', 'PDF', 'VIDEO'
    uploaded_at timestamptz DEFAULT NOW(),
    CONSTRAINT fk_material_session FOREIGN KEY (session_id) REFERENCES session(id) ON DELETE CASCADE
);
```

### Потоки данных

| Откуда | Куда | Что передаётся | Протокол |
|--------|------|----------------|----------|
| Frontend | Backend | Логин, создание сессий, материалы | REST + Cookie |
| Backend | БД | Пользователи, сессии, материалы | JPA/Hibernate |
| Backend | Frontend | JWT токен для Jitsi | REST |
| Frontend | Jitsi | Видео/аудио поток, JWT токен | WebRTC + JWT |
| Frontend | Backend | **НЕ передаёт медиа** | - |

**Важно:** 
- ✅ Frontend делает запросы **только к Backend** (REST API)
- ✅ Frontend подключается к **Jitsi напрямую** (IFrame API)
- ✅ Backend **не передаёт медиа** — только генерирует JWT токены
- ✅ Медиа (видео/аудио) идёт **напрямую между клиентом и Jitsi**

### Механизм работы

#### 1. Авторизация (куки не сохраняется, авторизация требуется каждый раз)
```
Frontend → POST /api/auth/login → Backend → БД
Frontend ← JSESSIONID (cookie) ← Backend
```

#### 2. Создание занятия
```
Frontend → POST /api/sessions → Backend → БД (INSERT session)
Frontend ← SessionDTO ← Backend
```

#### 3. Подключение к видеозвонку
```
1. Frontend → POST /api/sessions/{id}/join-token → Backend
2. Backend → генерирует JWT токен → Frontend
3. Frontend → Jitsi IFrame (с JWT токеном) → Jitsi Meet
4. Jitsi ← проверка JWT → подключение к комнате
```

#### 4. Демонстрация экрана
```
Frontend → Jitsi API (toggleScreenShare) → Jitsi
Jitsi → транслирует экран всем участникам → Jitsi
```

**Важно:** Медиа (видео/аудио/экран) **НЕ проходит через Backend** — только через Jitsi.

## Механизм работы видеозвонка

### 1. Логопед создаёт сессию

```
1. Логопед → Frontend: форма создания сессии
2. Frontend → Backend: POST /api/sessions
3. Backend → БД: INSERT session (room_name: "room-abc123")
4. Backend → Frontend: SessionDTO
```

### 2. Логопед подключается к сессии

```
1. Логопед → Frontend: кнопка "Подключиться"
2. Frontend → Backend: POST /api/sessions/{id}/join-token
3. Backend → генерирует JWT:
   {
     "iss": "logopedic_app",
     "aud": "logopedic_app",
     "sub": "1",
     "context": { "user": { "moderator": true } },
     "room": "room-abc123",
     "exp": 1773086082
   }
4. Backend → Frontend: { url, token, roomName }
5. Frontend → Jitsi IFrame: подключаемся с JWT токеном
6. Jitsi ← проверяет JWT ← Backend secret
7. Jitsi → Логопед: видеоконференция
```

### 3. Пациент подключается к сессии

```
1. Пациент → Frontend: кнопка "Подключиться"
2. Frontend → Backend: POST /api/sessions/{id}/join-token
3. Backend → проверка прав (пациент в session_patients)
4. Backend → генерирует JWT (moderator: false)
5. Frontend → Jitsi IFrame: подключаемся с JWT токеном
6. Jitsi → Пациент: видеоконференция
```

### 4. Демонстрация презентации

```
1. Логопед → Jitsi API: toggleScreenShare()
2. Jitsi → транслирует экран → все участники
3. Пациенты видят экран логопеда
```
Для отображение пациента можно использовать расширение браузера: Picture in Picture, оно отображает собеседника поверх всех окон 

Саму же презентацию отображать настроив:

- Слайд-шоу
- Настройка слайд-шоу
- Показ слайдов: управляемый пользователем (окно)

## Безопасность

### JWT токены Jitsi

| Параметр | Значение |
|----------|----------|
| Алгоритм | HS256 |
| Срок жизни | 2 часа |
| Secret | 32+ символа |
| Moderator | true (логопед) / false (пациент) |

### Проверка прав

1. **Backend проверяет:**
   - Пользователь авторизован (JSESSIONID)
   - Пользователь имеет доступ к сессии (логопед или пациент в списке)
   - Временное окно (15 минут до начала)

2. **Jitsi проверяет:**
   - JWT токен валиден (подпись)
   - Срок действия не истёк
   - Комната совпадает

## Диагностика проблем

### Дисконнекты через 1 минуту

**Проблема:** P2P нестабилен в Tailscale

**Решение:** Отключить P2P в `jitsi.service.ts`:
```typescript
p2p: { enabled: false }
```

### Нет видео/звука

**Проблема:** JVB не рекламирует правильный IP

**Решение:** В `jitsi/.env`:
```bash
JVB_ADVERTISE_IPS=200.87.154.47  # Ваш реальный IP
```

### CORS ошибки

**Проблема:** Frontend и Backend на разных доменах

**Решение:** Настроить CORS в `WebConfig.java`:
```java
.allowedOrigins("https://angular.cloudpub.ru")
.allowCredentials(true)
```

### 6. Настройить файрволл

Даже если Tailscale работает, Windows блокирует входящие подключения на порт 8443 с других устройств. Нужно создать правильное правило:

Шаг 1:
- Нажмите Win + R, введите wf.msc, нажмите Enter
- В левом меню выберите "Правила для входящих подключений"
- Справа нажмите "Создать правило..."
- Выберите "Для порта" → Далее

Настройте:

- TCP
- Определенные локальные порты: 8443
- Далее
- Выберите "Разрешить подключение" → Далее
- ВАЖНО! Оставьте галочки только на:
- ✅ Частная (Private)
- ✅ Доменная (Domain)
- Снимите галочку с "Публичная" (Public) — это важно для безопасности!
- → Далее
- Имя: Jitsi Meet Tailscale → Готово

Шаг 2: Создайте еще одно правило для UDP (если нужно)
- Повторите шаги, но на выборе протокола  выберите:
- UDP
- Порт: 10000 (это порт для видео Jitsi)
